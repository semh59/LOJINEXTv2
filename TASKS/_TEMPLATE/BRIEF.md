# BRIEF.md

## Task ID
TASK-XXXX

## Task Name
[One short phrase]

## Phase
[Number and name]

## Primary Purpose
The single most important thing that must exist after this task.
One sentence. If it needs two, split the task.

## Expected Outcome
Specific, verifiable conditions that mean this task succeeded.

Write like:
  "Endpoint X returns 201 with field Y"
  "User cannot do Z — returns 403"

Not like:
  "Feature X works"
  "Tests are written"

## In Scope
What will be built. Be specific about what, not just which area.

## Out of Scope
What will NOT be done. Prevents unilateral expansion.

## Dependencies
What must exist or be complete before this task can finish.

## Notes for the Agent
Prior decisions that apply, known traps, references, constraints.
