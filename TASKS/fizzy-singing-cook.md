## Confirmed Defects & Fixes

### DEFECT 1 — Migration column length drift (blocking for alembic check / schema sync)

**Files:** `alembic/versions/a1b2c3d4e5f6_trip_service_baseline.py`,
          `alembic/versions/d1e2f3a4b5c6_add_trip_audit_log.py`,
          `src/trip_service/models.py`

**Issue:** Baseline migration used `String(50)` for all ULID reference columns and `String(20)` for
actor_type columns. Audit log migration used `String(64)` for actor_id. Models use correct `String(26)`
for ULIDs and `String(32)` for actor_type/role fields. Running `alembic check` will fail; future
autogenerated migrations will emit spurious ALTER TABLE statements.

**Columns affected:**
| Column | Migration | Model | Correct |
|---|---|---|---|
| driver_id | String(50) | String(26) | String(26) |
| vehicle_id | String(50) | String(26) | String(26) |
| trailer_id | String(50) | String(26) | String(26) |
| route_pair_id | String(50) | String(26) | String(26) |
| route_id | String(50) | String(26) | String(26) |
| created_by_actor_id | String(50) | String(26) | String(26) |
| soft_deleted_by_actor_id | String(50) | String(50) | String(50) ✓ |
| actor_id (timeline) | String(50) | String(26) | String(26) |
| actor_id (delete_audit) | String(64) | String(26) | String(26) |
| created_by_actor_type | String(20) | String(32) | String(32) |
| actor_type (timeline) | String(20) | String(32) | String(32) |
| actor_role (delete_audit) | String(32) ✓ | String(32) | String(32) ✓ |

**Fix:** New corrective migration `e1f2a3b4c5d6_fix_column_lengths.py` with ALTER TABLE statements.
PostgreSQL `VARCHAR(50)→VARCHAR(26)` is safe when all existing data is ≤26 chars (all ULIDs are
exactly 26). `VARCHAR(20)→VARCHAR(32)` is an expansion, always safe.

```python
# upgrade
op.alter_column("trip_trips", "driver_id", type_=sa.String(26))
op.alter_column("trip_trips", "vehicle_id", type_=sa.String(26))
op.alter_column("trip_trips", "trailer_id", type_=sa.String(26))
op.alter_column("trip_trips", "route_pair_id", type_=sa.String(26))
op.alter_column("trip_trips", "route_id", type_=sa.String(26))
op.alter_column("trip_trips", "created_by_actor_id", type_=sa.String(26))
op.alter_column("trip_trips", "created_by_actor_type", type_=sa.String(32))
op.alter_column("trip_trip_timeline", "actor_id", type_=sa.String(26))
op.alter_column("trip_trip_timeline", "actor_type", type_=sa.String(32))
op.alter_column("trip_trip_delete_audit", "actor_id", type_=sa.String(26))
```

### DEFECT 2 — 6 router endpoints missing `response_model`

**File:** `src/trip_service/routers/trips.py`

**Issue:** FastAPI cannot validate response structure or generate accurate Swagger docs for these
endpoints. All 6 return `TripResource` via `_json_response()` but lack the decorator declaration.

**Lines:**
- `1070` — `GET /api/v1/trips/{trip_id}`
- `1095` — `PATCH /api/v1/trips/{trip_id}`
- `1240` — `POST /api/v1/trips/{trip_id}/approve`
- `1300` — `POST /api/v1/trips/{trip_id}/reject`
- `1338` — `POST /api/v1/trips/{base_trip_id}/empty-return`
- `1476` — `POST /api/v1/trips/{trip_id}/cancel`

**Fix:** Add `response_model=TripResource` to each `@router` decorator.
Note: Internal ingest endpoints (lines 443, 593, 709) use `_json_response()` intentionally
for ORJSONResponse performance — those do NOT need response_model.

### DEFECT 3 — `latest_evidence()` non-deterministic tiebreaker

**File:** `src/trip_service/trip_helpers.py` (line ~48)

**Issue:** Sort key is `created_at_utc` only. Two evidence records created in the same microsecond
(possible under test or bulk import) have non-deterministic ordering.

**Fix:** Add `id` as secondary sort key (ULID is lexicographically time-ordered, safe tiebreaker):

```python
return sorted(trip.evidence, key=lambda e: (e.created_at_utc, e.id), reverse=True)[0]
```

### DEFECT 4 — `.env.example` missing configurable TTL vars

**File:** `.env.example` (root of trip-service)

**Issue:** Two operationally important tuning knobs are absent from documentation:
- `TRIP_OUTBOX_RELAY_CLAIM_TTL_SECONDS` (default 60) — controls stale-claim recovery window
- `TRIP_IDEMPOTENCY_RETENTION_HOURS` (default 24) — controls idempotency record lifetime

**Fix:** Add both to `.env.example` with default values and comments.

### ETag Usage for Frontend

Frontend must implement optimistic locking:
1. `GET /api/v1/trips/{id}` → read `ETag` response header

3. 412 response → user must re-fetch and retry
