# TASK-0007: Approval Flow

## Purpose

Implement the normative approval flow for Route Pairs (Section 4.2). This allows promoting a `DRAFT` version (generated by the processing pipeline) to `ACTIVE` status, while automatically superseding any existing `ACTIVE` versions for that pair.

## Requirements

- Atomic database operation for version promotion.
- Update `RoutePair` version pointers (`current_active_forward_version_no`, etc.).
- Ensure `DRAFT` versions are moved to `ACTIVE` status.
- Expose via `POST /v1/pairs/{pair_id}/approve`.

## Success Criteria

- [x] Domain logic for atomic promotion implemented.
- [x] API endpoint integrated.
- [x] Verified with integration tests (mocked).
- [x] Handoff files created.
